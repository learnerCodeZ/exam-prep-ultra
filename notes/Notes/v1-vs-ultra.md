# 版本对比：v1.0 vs v2.0 vs v3.0

## 架构差异

| | v1.0 Classic | v2.0 Ultra 网页版 | v3.0 Ultra 全栈版 |
|--|-------------|-------|-------|
| **代码组织** | 单文件（index.html 649行，含全部 JS/CSS） | 分离（HTML + app.js + parser-md.js + CSS） | 分离 + 后端（HTML + JS 模块 + functions/ API） |
| **题库数据** | data.js 内嵌（620KB，写死在 JS 里） | default.json 外置 + localStorage | KV 存储 + D1 元数据 + localStorage 缓存 |
| **题库管理** | 无（只有一套固定题库） | 多题库：切换、重命名、删除、追加 | 多题库 + 云端题库（公开/私密） + 好友共享 |
| **解析器** | 无（Python 脚本离线生成 data.js） | 浏览器端实时解析 Word/PDF/MD/粘贴文本 | 同 v2.0 |
| **后端** | 无 | 无（纯静态 SPA） | Cloudflare D1 + KV + Pages Functions |
| **用户系统** | 无 | 无 | 注册/登录/登出，PBKDF2 密码哈希 |
| **部署** | 本地打开 index.html | GitHub Pages / Cloudflare Pages | Cloudflare Pages（Functions + D1 + KV） |

## 代码量对比

- v1.0：**22 个函数**（纯刷题渲染）
- v2.0：**55 个函数**（刷题 + 导入 + 题库管理 + 答案编辑 + 帮助）
- v3.0：前端 ~55 个函数 + 后端 ~15 个 API Handler

## 总结

- **v1.0**：只能刷固定题库的纯刷题工具，数据写死在代码里
- **v2.0**：从"看题工具"变成"题库平台"——加了完整的题库生命周期（导入→管理→刷题→编辑），但仍是纯静态
- **v3.0**：从"纯前端"变成"全栈应用"——加了用户系统、好友共享、管理面板，数据上云但保持离线兼容
