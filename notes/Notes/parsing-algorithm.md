# 题目解析算法

## 一、整体流程

所有文件最终都转成纯文本，走同一条解析管线：

```
Word (.docx)  ─→ mammoth.js 转文本 ─┐
PDF (.pdf)    ─→ pdf.js 转文本 ─────┼→ parseMarkdownBank() → 题目数组
Markdown / 粘贴文本 ─────────────────┘
```

- 文件转文本：`parser-docx.js`（Word）、`parser-pdf.js`（PDF）
- 文本转题目：`parser-md.js`（全部核心逻辑）

## 二、识别题型

### 2.1 有标题时（`# 听力`、`# 单选题` 等）

`detectType()` 按标题关键词映射：

| 标题含 | → 类型 |
|--------|--------|
| 听力 | listening |
| 段落匹配 | matching |
| 选词填空 | fillblank |
| 阅读 | reading |
| 单选 | singleChoice |
| 多选 | multiChoice |
| 判断 | judgment |

### 2.2 无标题时（粘贴的纯文本）

`autoDetectAndParse()` 按特征自动检测，优先级从高到低：

| 优先级 | 检测特征 | → 题型 |
|--------|---------|--------|
| 1 | `答案：对` / `答案：错` | 判断题 |
| 2 | `__X) word__` + `NUM)` 模式 | 选词填空 |
| 3 | `(X)` 答案标记 + 字母段落 | 段落匹配 |
| 4 | 标题含"多选" 或 答案多字母 | 多选题 |
| 5 | 题号 + ABCD 选项 | 单选题 |
| 6 | 题号 + ) + 问句 | 阅读 |
| 7 | 都不匹配 | 听力（兜底） |

混合内容（判断题+选择题在一段文本里）时，先按中文段落标题（`一、判断题`、`（一）多选题`）分节，每节独立走上述检测。

## 三、拆题逻辑

每种题型的拆法都是**正则逐行扫描**，状态机模式：遇到题号开新题，遇到选项/答案填充当前题。

### 3.1 单选题 / 多选题

```
1. 题干（  ）          ← 题号行：开新题
   A. 选项A             ← 选项行：加选项
   B. 选项B
   C. 选项C
   D. 格雷码正确答案：CD  ← 内联答案：从选项文本剥离
答案：C                 ← 独立答案行：记答案
```

关键正则：
```js
/^(\d+)[\.、．]\s*(.+)/              // 题号
/^([A-O])[\.\)、．]\s*(.+)/          // 选项
/^(?:正确)?答案[：:]\s*([A-O]+)/i     // 答案（单字母或多字母）
/^(.+?)(?:正确)?答案[：:]\s*([A-O]+)$/i // 内联答案剥离
```

单选和多选用同一个逻辑，区别只在答案长度（单选1个字母，多选2+个）。

### 3.2 判断题

逐行扫描同单选题，但：
- 答案 `对/√/✓/T/是` → `answer: 'A'`
- 答案 `错/×/✗/F/否` → `answer: 'B'`
- 自动生成固定选项 `[{A: '对'}, {B: '错'}]`

### 3.3 听力

同单选题，但题干固定为"第N题"（不读 stem 字段）。

### 3.4 段落匹配

两阶段扫描：
1. **收集段落**：`A. 段落文本` 存入 passages 对象
2. **收集题目**：`1. 题干 (X)` 提取题干和答案字母

选项列表 = passages 的所有 key（A、B、C…）。

### 3.5 选词填空

1. 找含多个 `A) word` 的行 → 词库（wordBank）
2. 词库前面的文本 → 文章（passage）
3. 从文章中提取 `__X) word__ NUM)` → 空白映射（blanks：`{"26": "C"}`）

### 3.6 阅读

按 `## Passage` 标题分篇，每篇内按题号拆题，选项走通用 `parseOptions()`。

## 五、局限

- Word 双栏排版丢右栏（mammoth 按段落顺序输出，右栏内容丢失）
- 纯规则匹配，格式不规范就解析不出来
- 没有做 NLP / AI 识别，全靠字符串匹配
