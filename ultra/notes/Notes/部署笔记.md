# 部署笔记 — Wrangler、部署流程与种子数据

---

## 一、Wrangler 是什么

### 一句话解释

Wrangler 是 Cloudflare 官方的命令行工具，用来在本地开发、管理资源、部署项目到 Cloudflare。

### 类比理解

你用过 Git 吗？Git 管理代码版本，命令行工具是 `git`。
Wrangler 管理的是 Cloudflare 上的资源（数据库、存储、网站），命令行工具是 `npx wrangler`。

| Git | Wrangler |
|-----|----------|
| `git init` 初始化仓库 | `wrangler pages project create` 创建项目 |
| `git push` 推到远程 | `wrangler pages deploy` 部署到云端 |
| `git pull` 拉取代码 | `wrangler d1 execute` 操作数据库 |
| 代码存在 GitHub | 网站存在 Cloudflare |

### 为什么用 `npx wrangler` 而不是 `wrangler`？

`npx` 是 Node.js 自带的工具，可以临时下载并运行一个 npm 包，不需要全局安装。所以：
- `npx wrangler login` = 临时下载 wrangler，然后执行 login
- 好处：不用 `npm install -g wrangler`，不会污染全局环境

### 常用 Wrangler 命令

```bash
# 登录 Cloudflare 账号（会弹浏览器授权）
npx wrangler login

# 查看当前登录状态
npx wrangler whoami

# 创建 D1 数据库
npx wrangler d1 create 数据库名

# 执行 SQL 文件（建表/插入数据）
npx wrangler d1 execute 数据库名 --remote --file=schema.sql

# 创建 KV 命名空间（类似一个键值对存储桶）
npx wrangler kv namespace create 命名空间名

# 往 KV 写入一条数据
npx wrangler kv key put --namespace-id=xxx --remote "键名" --path=本地文件路径

# 本地开发服务器（模拟 Cloudflare 环境）
npx wrangler pages dev .

# 部署到 Cloudflare Pages
npx wrangler pages deploy . --project-name=项目名 --branch=main
```

### `--remote` 是什么意思？

- 不加 `--remote`：操作的是**本地**模拟环境（存在你电脑上）
- 加 `--remote`：操作的是**云端**真实环境（存在 Cloudflare 服务器上）

生产数据必须加 `--remote`，否则数据只写到你本地，线上访问不到。

## 二、部署流程详解

### 整体流程图

```
登录 Cloudflare
    │
    ▼
创建云端资源（D1 数据库 + 2 个 KV 命名空间）
    │
    ▼
填入 wrangler.toml（告诉项目用哪些资源）
    │
    ▼
建表（schema.sql → D1）
    │
    ▼
种子数据（seed.sql → D1）
    │
    ▼
上传默认题库（default.json → KV）
    │
    ▼
部署代码到 Cloudflare Pages
    │
    ▼
线上访问 ✅
```

### 每一步做了什么

#### 1. 登录

```bash
npx wrangler login
```

弹浏览器，点授权。成功后 wrangler 记住你的身份，后续命令都带着你的权限。

#### 2. 创建云端资源

Cloudflare 提供三种存储，我们的项目用了两种：

| 资源 | 用途 | 类比 |
|------|------|------|
| **D1** | 关系型数据库，存用户/好友/题库元数据 | MySQL |
| **KV** (SESSION) | 存登录 session（token → 用户ID） | Redis |
| **KV** (QUESTIONS) | 存题目的 JSON 大文件 | 对象存储 |

创建命令：
```bash
npx wrangler d1 create exam-prep-db          # 创建数据库
npx wrangler kv namespace create SESSION      # 创建 session 存储
npx wrangler kv namespace create QUESTIONS    # 创建题目存储
```

每个命令执行后会输出一个 ID，类似 `e7c2a462-2b73-468b-8c4a-e1258509d3f5`，这个 ID 是资源的唯一标识。

#### 3. 填入 wrangler.toml

`wrangler.toml` 是项目的配置文件，告诉 Cloudflare："我的项目要用哪些资源"。

```toml
# 顶层：本地开发用
[[d1_databases]]
binding = "DB"              # 代码里用 env.DB 访问
database_name = "exam-prep-db"
database_id = "e7c2a462-..."  # ← 第2步获得的 ID

[[kv_namespaces]]
binding = "SESSION"          # 代码里用 env.SESSION 访问
id = "95a216..."

[[kv_namespaces]]
binding = "QUESTIONS"        # 代码里用 env.QUESTIONS 访问
id = "966124..."

# env.production：线上部署用（必须重复声明，否则线上绑定会丢失！）
[[env.production.d1_databases]]
binding = "DB"
database_name = "exam-prep-db"
database_id = "e7c2a462-..."

[[env.production.kv_namespaces]]
binding = "SESSION"
id = "95a216..."

[[env.production.kv_namespaces]]
binding = "QUESTIONS"
id = "966124..."
```

**关键踩坑**：必须同时声明顶层和 `env.production`，否则部署时绑定会丢失，线上 `env.DB` 就是 undefined。

#### 4. 建表

```bash
npx wrangler d1 execute exam-prep-db --remote --file=schema.sql
```

把 `schema.sql` 里的 SQL 发到云端 D1 执行，创建 3 张表：users、friends、banks。

类比：新买了一个 MySQL 数据库，先 `CREATE TABLE` 建好表结构。

#### 5. 种子数据

```bash
npx wrangler d1 execute exam-prep-db --remote --file=seed.sql
```

往空表里插入最初始的数据（管理员账号 + 默认题库记录）。详见下一节。

#### 6. 上传默认题库

```bash
npx wrangler kv key put --namespace-id=966124... --remote "banks/default.json" --path=data/default.json
```

把本地 `data/default.json`（700KB，191道题）上传到云端 KV，key 是 `banks/default.json`。

这样线上用户打开网站时，代码通过 `env.QUESTIONS.get("banks/default.json")` 就能读到题目。

#### 7. 部署

```bash
npx wrangler pages project create exam-prep-ultra --production-branch=main  # 首次：创建项目
npx wrangler pages deploy . --project-name=exam-prep-ultra --branch=main --commit-dirty=true  # 部署
```

把当前目录所有文件上传到 Cloudflare，自动编译 Pages Functions（后端 API），生成访问地址：
👉 `https://exam-prep-ultra.pages.dev/`

`--commit-dirty=true` 是因为我们没有 git commit，加这个跳过"未提交文件"的警告。

### 为什么原来用 R2 后来改 KV？

R2 是 Cloudflare 的对象存储，需要绑卡才能用。KV 不需要绑卡，免费额度也够用。

| 对比 | R2 | KV |
|------|-----|-----|
| 需要绑卡 | ✅ 需要 | ❌ 不需要 |
| 单值大小限制 | 5TB | 25MB |
| 读取方式 | `await obj.text()` 两步 | 直接返回字符串 |
| 适合场景 | 大文件（视频等） | 键值对、小文件 |

我们的题目 JSON 最大 700KB，KV 完全够用。

## 三、种子数据是什么

### 一句话解释

种子数据就是"初始数据"——建完数据库后，往空表里插入的最基本的数据，让系统第一次启动就能用。

### 类比理解

- 买了新手机 → 开机后有默认壁纸、默认铃声 = 种子数据
- 装了新游戏 → 有初始角色、初始金币 = 种子数据
- 建了新数据库 → 有管理员账号、默认题库 = 种子数据

### 我们的种子数据

```sql
-- 1. 管理员账号
INSERT OR IGNORE INTO users (email, password, nickname, role)
VALUES ('admin@example.com', '哈希值...', '管理员', 'admin');

-- 2. 默认题库记录
INSERT OR IGNORE INTO banks (id, owner_id, name, is_default, is_public, kv_key)
VALUES (1, 1, '大学英语4（默认）', 1, 1, 'banks/default.json');
```

#### 为什么需要管理员账号？

- 没有管理员，就无法登录管理面板
- 管理员是第一个用户（id=1），新用户注册时自动和管理员加好友
- 密码不能用明文存，必须哈希后存

#### 管理员密码哈希是怎么生成的？

```bash
node scripts/gen-admin-hash.js admin123
# 输出：8cECt69wvlVRL3Y+4hvxLQ==:pHc1HUC6qQHyxMFOdRFiYLNg8f4eQp5MSbPOYXu95KU=
```

这个脚本做的事：
1. 生成 16 字节随机 salt
2. 用 PBKDF2-SHA256 算法对密码做 10 万次迭代
3. 输出 `salt:hash` 格式的字符串

然后把这个哈希值粘贴到 seed.sql 里，插入数据库。

**为什么不直接在 seed.sql 里写明文密码？** 因为数据库里存的就是哈希，登录时用同样的算法算哈希再比对，永远不存明文。

#### 为什么需要默认题库记录？

- 数据库的 banks 表需要有一条记录，指向 KV 中的 `banks/default.json`
- `is_default = 1` 表示这是系统自带的默认题库
- `kv_key = 'banks/default.json'` 是 KV 中的 key，代码通过这个 key 去读题目

#### `INSERT OR IGNORE` 是什么意思？

- `INSERT`：插入数据
- `OR IGNORE`：如果这条数据已经存在（主键冲突），就跳过不报错

好处：seed.sql 可以重复执行，不会因为"管理员已存在"而报错。

### 种子数据 vs 用户数据

| | 种子数据 | 用户数据 |
|--|---------|---------|
| 什么时候产生 | 部署时一次性插入 | 用户注册/使用时产生 |
| 存在哪里 | seed.sql 文件 | 数据库 |
| 谁创建的 | 开发者 | 用户自己 |
| 例子 | 管理员账号、默认题库 | 普通用户、好友关系、用户创建的题库 |

## 四、如果需要重新部署

以后改了代码，只需要一条命令：

```bash
cd ultra
npx wrangler pages deploy . --project-name=exam-prep-ultra --branch=main --commit-dirty=true
```

如果改了数据库结构，需要重新建表：

```bash
# 先删除旧表（会丢数据！生产环境慎用）
npx wrangler d1 execute exam-prep-db --remote --command="DROP TABLE IF EXISTS banks; DROP TABLE IF EXISTS friends; DROP TABLE IF EXISTS users;"

# 重建
npx wrangler d1 execute exam-prep-db --remote --file=schema.sql
npx wrangler d1 execute exam-prep-db --remote --file=seed.sql
```

如果只改了种子数据（比如换管理员密码）：

```bash
# 重新生成哈希
node scripts/gen-admin-hash.js 新密码

# 替换 seed.sql 中的哈希值后重新执行
npx wrangler d1 execute exam-prep-db --remote --file=seed.sql
```
