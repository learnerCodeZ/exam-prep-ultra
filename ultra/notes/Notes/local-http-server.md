# 什么是 HTTP 服务

## 一句话解释

HTTP 服务就是一个程序在某个端口监听，浏览器发请求过来，它把对应的文件返回去。本质上就是一个**文件搬运工**。

## 类比理解

想象一个图书馆：

- 你（浏览器）走到前台说："我要借 `/src/index.html`"
- 图书管理员（HTTP 服务）去书架上找到这本书，递给你
- 你又说："我还要 `/css/style.css`"，管理员再去找来给你
- 每次你要一个文件，就发一次请求，管理员就回一次响应

这个过程就是 HTTP 协议：**请求 → 响应**，一问一答。

## 我们项目里的用法

```bash
cd d:/MYCODE/E/ultra
python -m http.server 8765
```

这行命令用 Python 自带的模块起了一个最简单的 HTTP 服务：

- 把 `ultra/` 目录当成"书架"（网站根目录）
- 在 8765 端口"值班"
- 浏览器访问 `http://localhost:8765/src/index.html`，它就把这个文件发过去
- 只在本机可见，关掉终端就停了

## 为什么不能直接双击 index.html

双击打开用的是 `file://` 协议，浏览器出于安全限制会拦截 `fetch()` 请求（比如读取 `data/default.json`），导致题库加载失败。用 HTTP 服务就和真实部署环境一致，`fetch` 能正常工作。
