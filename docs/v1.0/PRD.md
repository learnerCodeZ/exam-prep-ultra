# PRD — 期末刷题宝典 Ultra

## 1. 产品概述

一个面向所有学生的在线刷题工具。用户打开网址即可刷题，也可导入自己的题库（Word/Markdown/PDF）。零注册、零安装、零服务器成本。

## 2. 目标用户

- 需要期末复习的大学生
- 任何想用"刷题"方式记忆知识的人

## 3. 核心场景

1. 打开网址 → 选题型 → 开始刷题 → 即选即判 → 查看正确答案及翻译
2. 导入自己的题库文件 → 自动解析 → 开始刷题
3. 错题自动收录 → 反复刷错题直到全对

## 4. 功能需求

### 4.1 刷题功能（继承 v1.0）

| 功能 | 说明 |
|------|------|
| 题型切换 | 听力、段落匹配、选词填空、阅读，Tab 切换 |
| 四种模式 | 顺序练习、随机刷题、错题本、收藏夹 |
| 即选即判 | 选择后立即显示对错 + 正确答案 + 翻译 |
| 题目导航 | 点击题号弹出面板，跳转任意题目 |
| 答题统计 | 总题数、已答对、错题数、收藏数 |
| 数据持久化 | localStorage 保存答题记录、错题、收藏 |

### 4.2 题库管理（新增）

#### 4.2.1 默认题库

- 从 npoint.io 拉取官方题库 JSON
- 只读，所有用户共享
- 管理员（项目所有者）本地生成 JSON 后手动更新 npoint

#### 4.2.2 导入题库

用户点击"导入题库"按钮，选择本地文件，浏览器端解析后存入 localStorage，立即可用。

| 格式 | 方案 | 说明 |
|------|------|------|
| Markdown (.md) | 纯文本正则解析 | 支持标准格式（见 4.2.3） |
| Word (.docx) | mammoth.js | 浏览器端读取 docx 提取文本 |
| PDF (.pdf) | pdf.js | 浏览器端提取文本 |

#### 4.2.3 Markdown 题库格式

```markdown
# 听力

1.
A. Option text
B. Option text 中文翻译
C. Option text
D. Option text

2.
A. Option text
B. Option text
C. Option text 中文翻译
D. Option text

# 段落匹配

1.

A. Passage text...
B. Passage text...
C. Passage text...

1. Question stem (A)
2. Question stem (B)

# 选词填空

## Unit 1

Passage text with __B) word__ 31) blanks...

A) word1   B) word2   C) word3   ...

# 阅读

## Passage 1

41) Question stem?
A. Option
B. Option 中文翻译
C. Option
D. Option
```

规则：
- 中文翻译紧跟选项文本，表示该选项为正确答案
- 题型用一级标题 `#` 标记
- 段落匹配中段落用字母+点号标记，题目末尾括号内为答案
- 选词填空中 `__LETTER) word__ NUMBER)` 标记空格和答案

#### 4.2.4 题库切换

- 顶部提供题库切换下拉菜单
- 选项：默认题库 + 用户导入的题库列表
- 导入的题库可删除

### 4.3 答案编辑（新增）

- 刷题时，每题提供"编辑答案"按钮
- 点击后弹出编辑面板，可修改当前题的答案
- 修改仅保存在 localStorage，不影响 npoint 上的官方题库
- 已修改答案的题目显示小标记（如铅笔图标）

## 5. 技术架构

```
┌─────────────────────────────────────┐
│           GitHub Pages              │
│        (静态网站托管, 免费)           │
│                                     │
│  index.html + CSS + JS (SPA)        │
│  ├─ mammoth.js   (Word 解析)        │
│  ├─ pdf.js       (PDF 解析)         │
│  └─ 本地 JS       (MD 解析)          │
└──────────┬──────────────────────────┘
           │ GET (只读)
           ▼
┌─────────────────────────────────────┐
│           npoint.io                 │
│      (JSON 存储, 免费)               │
│                                     │
│  默认题库 JSON (管理员手动更新)       │
└─────────────────────────────────────┘

┌─────────────────────────────────────┐
│        浏览器 localStorage           │
│                                     │
│  ├─ 答题记录 (对/错)                │
│  ├─ 错题本                          │
│  ├─ 收藏夹                          │
│  ├─ 用户导入的题库                   │
│  └─ 答案修改记录                     │
└─────────────────────────────────────┘
```

### 5.1 技术选型

| 层 | 选型 | 理由 |
|----|------|------|
| 前端 | 原生 HTML/CSS/JS | 轻量、无构建、直接部署 |
| Word 解析 | mammoth.js | 浏览器端 docx → 文本，成熟稳定 |
| PDF 解析 | pdf.js (Mozilla) | 浏览器端 PDF 文本提取，业界标准 |
| MD 解析 | 自写正则 | 题库格式简单，无需重型解析器 |
| 题库存储 | npoint.io | 免费、公开只读、JSON 直出 |
| 本地存储 | localStorage | 答题记录、导入题库、答案修改 |
| 部署 | GitHub Pages | 免费、CDN、自动 HTTPS |

### 5.2 数据流

```
默认题库:  npoint.io GET → 前端渲染
导入题库:  本地文件 → mammoth/pdf.js/正则解析 → localStorage → 前端渲染
答题记录:  用户操作 → localStorage
答案修改:  用户编辑 → localStorage (覆盖默认答案)
```

## 9. 风险与缓解

| 风险 | 缓解 |
|------|------|
| npoint.io 宕机 | 前端内置 fallback JSON；首次加载后缓存到 localStorage |
| npoint.io 写入无保护 | 题库只读下发，管理员手动更新，用户无法直接改 npoint |
| localStorage 容量限制（5-10MB） | 导入题库过多时提示清理；优先存储答题记录 |
| Word/PDF 格式千差万别 | 解析后提供预览编辑步骤，用户可手动修正再确认 |
